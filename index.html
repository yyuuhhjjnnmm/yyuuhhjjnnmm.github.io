Nielsen: Global Connect

Software Developer

At Nielsen, I worked in the Canadian Technology Department, on the team responsbile for rewriting the old core calculation engine. While on that team, I contributed to various parts of the project:

1. Data Preprocessing Program
This application took the existing underlying data pipeline and eliminated the numerous files from the old workflow. This was achieved by aggregating the data into a single JSON file to be taken as input by the new calculation engine. Due to the C++ dominate workflow, the POCO JSON library was used to format the resulting JSON. Within the preprocessing, additional logic was added in order to streamline the downstream calcualtion and optimize process time.

2. New Core Calculation Engine
The new calculation engine was designed to replace the old engine's functionaility, but improving the design with Object-Oriented design and leveraging the Curiously Recurring Template Pattern in order to reduce the runtime cost of by implementing static polymorphism.

In order to benchmark the performance, we used linux tools such as perf stat and gprof to benchmark the CPU time and to get a breakdown of hotspots. Simiialry, using Visual Studio 2019 disagonitc tools, the CPU usage and CPU time was used to get a benchmark of performance.

3. Python Scripts
On the Linux servers, we developed and maintained Python scripts in order to run test orders for testing the complete order flow, the correctness of our new calculation engine compared to the old, and also to benchmark the performance of our new engine.

4. Sybase Investigation
During development, we realized that even though our intermediary results matched with the old engine, our end to end testing showed discrepencies. I was tasked with tracking down the source of this error. I proceeded by manually running all the programs in the pipeline and comparing intermediary Sybase tables using SQL. Here I was able to backtrace the root cause of the mismatch, and by investigating the SQL stored-procedures of the downstream programs, I was able to determine the reason for the problem. One of the files that the old engine was producing was actually required by downstream programs, thus we implemented the required functionailty to support that file and were able to resolve this problem.

Scotiabank

Software Developer

At Scotiabank, I worked in the Global Banking and Markets (GBM) division, with the Global Analytics and Financial Engineering (GAFE) team.

During my time with GAFE, I worked on two distinct projects:
1. Develop an Object Oriented data handling system in Python
Due to new ISDA regulations, Scotibank needed to a new methodology in order to comply with the Standard Initial Margin Model (SIMM). Previously, a proof-of-concept was developed for the core calculations. In order to test the existing solution, the GAFE team was tasked to develop a pipeline for the data flowing through, extract relevant trade information and to feed the subset of data through the calculator.

This aggregation and scripting was done through populating our own Object-Oriented data structure with Python scripts. Our group used BitBucket for revision control and peer review. Throughout the project, our team was able to reduce the processing time of over one million trade entries by more than 400% by minimizing I/O to file, while still considering space, time and security tradeoffs.

2. Develop new portfolio functionality in C++
One of the softwares used by internal traders at Scotiabank is IBM Algo One. This is an enterprise software solution developed by IBM. The core code of the software is developed in C++ and is an Object-Oriented Program.

One of the departments requested GAFE to integrate new portfolio functionalities for the software. Thus I was tasked with developing code to handle new portfolio dynamics. Alongside the implementation, I also wrote Bash Shell scripts to retrieve archived market data for testing. Furthermore, I added a new test suite to the existing daily overnight regression testing system. Finally, to accompany the changes, I composed a technical manual detailing sample useage and implementation specifics.
